Positions

1st Content

Static Position : This is the default position value for all HTML elements.

2nd Content

Relative Position: When you apply relative positioning to an element and then use the top, bottom, left, or right properties, the element will be moved from its original position without affecting the layout of other elements.

3rd Content

Absolute Position: An element with absolute positioning is positioned relative to the viewport, which means it always stays in the same place even if the page is scrolled.The body tag is by default relative

4th content

Sticky Position:Sticky positioning is a hybrid of relative and fixed positioning. A sticky positioned element behaves like a relatively positioned element until it reaches a specified point (defined by the top, right, bottom, or left properties), at which point it becomes fixed. Sticky positioning is commonly used for headers that stick to the top of the viewport when scrolling down a page.

5th Content

Fixed Position : Fixed positioning is similar to absolute positioning, but the element is fixed relative to the viewport, meaning it stays in the same place even when the page is scrolled. Fixed elements are often used for headers, footers, or navigation bars that should remain visible as the user scrolls through the page
